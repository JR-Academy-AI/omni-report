# JR Academy 每周增长玩法 — PRD

## 1. 背景与目标

### 1.1 为什么要做

JR Academy 现有的 marketing 自动化 routine 解决了 4 个问题：

- 竞品周报：**外部在做什么**
- 内容选题包：**讲座/活动/联名要做什么**
- AI 可见度：**SEO/GEO 数据**
- SEO 健康：**站点本身**

但 marketing 还有一个核心需求没被覆盖：**怎么用创意拉新和转化**。讲座是讲师/教研主导，**抽奖、裂变、游戏、限时活动这种纯运营驱动的玩法，需要持续的 idea 输入**。

现状：运营每周想玩法靠拍脑袋，看到瑞幸 1 元首单火了想抄一下，看到拼多多砍价火了又想抄一下——但没有结构化的 idea 库，也没有按"匠人业务 × 经典 growth hack"做匹配。

### 1.2 目标

每周自动产出 **5 个为 JR 业务量身设计的增长玩法 idea**，给运营/marketing 团队当周选 1-2 个去执行。

每个 idea 必须包括：
- 玩法名称（例："Cursor 7 天闯关挑战"）
- 一句话描述
- 灵感来源（如哪个经典 growth hack 改造的）
- 对应的 JR 业务（哪个 Bootcamp/Lab/产品）
- 目标用户（拉新 / 激活 / 留存 / 付费转化）
- 执行难度（🔴大 / 🟡中 / 🟢小）
- 预估指标（如：1 周招 100 人 / CAC 降低 30%）
- 落地步骤（3-5 步）

### 1.3 非目标

- ❌ 不替代讲座/线下活动选题（已被内容选题包覆盖）
- ❌ 不做单一玩法的深度方案（给 idea + 落地步骤即可，深方案运营再搞）
- ❌ 不直接落地（给清单，决策仍人做）

## 2. 输入数据源

| 来源 | 用途 |
|---|---|
| **竞品周报最新份** | 看竞品在做什么活动/折扣/裂变 → 借鉴或差异化 |
| **内容选题包最新份** | 知道本周市场热点 → 玩法蹭热点 |
| **weekly-holidays 数据** | 节日窗口（母亲节/EOFY/双 11/毕业季）→ 节日营销 idea |
| **JR 业务清单**（prompt 内嵌） | Bootcamp（AI Adoption / AI Engineer / AI PM / Vibe Coding 等）/ Lab / 求职服务 / 学员社区 |
| **经典 growth hack 案例库**（prompt 内嵌） | 灵感池：拼多多砍价、瑞幸 1 元、Duolingo streak、Notion 模板、Cursor 学生计划等 |

### 5.1 文件路径
```
growth-playbook/{YYYY-MM-DD}.md
```

### 5.2 Markdown 结构

```markdown
# JR 本周增长玩法 5 选 YYYY-MM-DD

## 总览
- 本周环境：[一句话总结，例：母亲节 + GPT-5.5 发布 + 竞品 Maven 30% off]
- 推荐优先级：① > ② > ③ > ④ > ⑤
- 涉及业务线：AI Engineer / Vibe Coding / Lab

---

## 玩法 ① **Cursor 7 天闯关挑战**（裂变 + 游戏）

| 字段 | 值 |
|---|---|
| **一句话描述** | 用户报名后 7 天每天解锁一个 Cursor 实战题，全部完成抽 5 名免费 Vibe Coding Bootcamp |
| **灵感来源** | Duolingo streak + 知乎答题瓜分 |
| **对应业务** | Vibe Coding Bootcamp 招生 |
| **目标用户** | 拉新（流量入口）+ 激活（试用 Cursor） |
| **执行难度** | 🟡 中（需要 7 道题 + 简单后端跟踪） |
| **预估指标** | 1 周报名 500 人，转化 5% = 25 人付费试听 |
| **本周做？** | ✅ 推荐本周启动（蹭 GPT-5.5 热度，14 天后母亲节大促前出结果） |

**落地步骤**：
1. **D-3**：教研出 7 道题（Cursor 入门到中级），后端做打卡页面
2. **D0**：小红书/B 站发起活动，发限时报名链接
3. **D1-7**：每天解锁 1 题，自动给完成者发奖
4. **D8**：抽奖直播，宣布结果
5. **D8+**：给所有完成者发优惠券，转化付费课程

## 玩法 ② **AI Engineer 简历互评瓜分** ...
（同结构）

## 玩法 ③ ④ ⑤ ...

---

## 这周不推的玩法（也列 1-2 个，说明为什么不适合）
- "拼团 5 人砍价 50%" — 我们客单价 ¥3000-5000，不像拼多多日用品适合砍价模式
- "学生半价" — 已有学生定价，再做活动会冲突
```

### 5.3 反 AI 味格式
- 禁止 "在当今...""综合性""值得关注""把握机遇" 等空话
- 每个 idea 必须具体到能写 PRD（"Cursor 7 天闯关，Day 1 教 chat，Day 2 教 inline edit..."），不是 "做一个学习游戏"
- 预估指标必须是数字（"1 周 500 报名"），不是 "高效拉新"
- 落地步骤必须 3-5 步可操作

## 6. 技术方案

### 6.1 配置

| 项 | 值 |
|---|---|
| Routine name | `JR Growth Playbook Weekly` |
| Cron | `0 21 * * 1` UTC = **周二 7am Brisbane** |
| Repo | `https://github.com/JR-Academy-AI/omni-report` |
| Model | `claude-sonnet-4-6` |
| Tools | Bash / Read / Write / Edit / WebSearch |
| MCP | Notion（hub TBD） |

### 6.2 跑通逻辑（防 stream idle timeout）

每周 5 个 idea，单个 idea 平均 ~150 tokens，整体 ~1500 tokens。可以一次写完，但保险起见用 3 Phase：

- **Phase 0**：读 PRD + 上游报告（竞品周报最新份 / 选题包最新份）
- **Phase 1**：写空骨架（5 个玩法 placeholder + 总览） + commit
- **Phase 2**：1 batch 一次写 5 个玩法 + 1 个不推荐玩法 → Edit + commit
- **Phase 3**：Edit 总览（环境 + 优先级） + commit "finalize"
- **Phase 4**：Notion 全文同步（一步法，沿用其他 routine pattern）

## 7. 验收标准

第一份必须满足：

- [ ] `growth-playbook/$DATE.md` 已 push（4-5 commit）
- [ ] 5 个玩法 idea，覆盖 3+ 业务线（不能 5 个都是 Vibe Coding）
- [ ] 每个 idea 字段齐全（描述 / 来源 / 业务 / 难度 / 指标 / 步骤）
- [ ] 至少 1-2 个不推荐玩法（说明原因）
- [ ] 0 条 AI 味开场
- [ ] Notion 页面创建且含全文

## 8. 一周节奏（含本 routine）

```
周日 20:00 — 竞品周报           (外部市场)
周一 07:00 — 内容选题包 #1      (本周讲座/活动)
周二 07:00 — 增长玩法 ✨        (本周创意池)  ← 新加
周三 04:00 — AI 可见度          (SEO/GEO 数据)
周三 07:00 — 内容选题包 #2      (周中调整)
```

**为什么周二**：周一 marketing 已知本周内容主题，周二补玩法 idea 池。运营周三可以从中挑 1-2 个开始落地。
